Burning Rubber

Burning Rubber is a great new app for creating and racing buggies! The simple to use create a buggy form allows you to choose from a number of different options to make your buggy the fastest it can be! Don't worry about the race rules, as your buggy can only be saved if it follows them. Once you've made your buggy, you can view and edit it to your hearts desire. Maybe your desire is to own a multitude of buggies, you can do that too! Add and delete buggies at any point, to ensure your fleet is the strongest it can be.

This easily took the most of my time during this project. This task could be split into three parts, adding new buggies to the database, being able to update different buggies and swaap between buggies.

To create a new buggy in the "new" route, I added a command that tells the database to insert a new set of data rather than updating the previous data. Then to show the buggy I went into the "buggy" route and told the code to fetch all of the data, instead of just one set. In the show buggy page I then looped through all the buggies to show all saved buggies.

To edit a previously created buggy I started by creating a button in the show buggy HTML linking to the page /edit/buggy_id. From here I went into the python and created a new route for /edit/buggy_id. In this route I made sure that it returned the buggy form page with the data from that given buggy using the value attribute on the buggy form.

I came up with a few errors in this task, the main problem being sending the correct costs assosiated with each buggy. To do this I created a dictionary with the costs of each buggy, which then got appended into a list and sent to the database along with the selections.
